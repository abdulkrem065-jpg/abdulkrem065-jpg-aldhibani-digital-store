# 🗺️ خارطة الحوكمة المعمارية (QEOS Governance Roadmap)

تُحدد هذه الوثيقة خارطة التشغيل الرسمية لجميع الوثائق المعمارية في نظام QEOS، وتفرض الترتيب الإلزامي لاستخدامها أثناء دورة حياة تطوير النظام.

---

## 1. التسلسل الرسمي للوثائق (Execution Order) ومتى يتم الرجوع إليها

يجب قراءة واستخدام الوثائق بالترتيب التشغيلي التالي:

1. **`QEOS_CONSTITUTION.md`**: يُقرأ أولاً في أي مهمة. يُرجع إليه لفهم الطبقات العشرين، حدود النظام، والمبادئ السيادية.
2. **`QEOS_DEVELOPMENT_CONSTITUTION.md`**: يُقرأ ثانياً. يُرجع إليه لمعرفة الممنوعات، قواعد التطوير، والقيود الصارمة قبل التنفيذ.
3. **`QEOS_SOVEREIGN_REGISTRY.md`**: يُرجع إليه للتحقق من صلاحية أي وثيقة أخرى وهل هي معتمدة أم مؤرشفة.
4. **الوثائق التشخيصية (`BASELINE.md`, `AUDIT_REPORT.md`)**: يُرجع إليها لفهم حالة المشروع الحالية ونقطة البداية.
5. **وثائق الهجرة (`MIGRATION_PLAN.md`, `MIGRATION_EXECUTION_MANIFEST.md`, `VERIFICATION_PROTOCOL.md`)**: يُرجع إليها حصراً عند تنفيذ موجات نقل وهيكلة المشروع.
6. **سجل القرارات (`docs/architecture/adr/`)**: يُرجع إليه عند الحاجة لفهم سبب اختيار تقنية معينة، أو عند اقتراح تغيير معماري جديد.

---

## 2. المرجع الأعلى عند التعارض
في حال حدوث أي تعارض هندسي، تقني، أو توثيقي بين أي ملفين في المشروع، فإن **`QEOS_CONSTITUTION.md`** هو المرجع الأعلى والمطلق، وتُلغى أي قاعدة تخالفه فوراً.

---

## 3. علاقة الوثائق ببعضها (Document Relationship Chain)

يتدفق مسار السلطة والتوثيق في المشروع وفق السلسلة التالية:
**Constitution** (يضع القوانين العليا) 
→ **Development Constitution** (يضع قوانين التنفيذ) 
→ **Baseline** (يحدد نقطة الصفر الحالية) 
→ **Audit Report** (يكتشف الفجوات) 
→ **Migration Plan** (يضع استراتيجية الحل) 
→ **Migration Manifest** (يحدد خطوات التنفيذ) 
→ **Verification Protocol** (يضمن نجاح التنفيذ) 
→ **ADRs** (يسجل أي قرارات جديدة بعد استقرار النظام).

---

## 4. دورة العمل الرسمية (Official Workflow)

تبدأ الدورة من الفكرة وحتى الدمج الفعلي (Integration) في النظام:

1. **الفكرة/الطلب (Ideation):** استلام متطلب جديد أو مقترح تغيير هندسي.
2. **المقترح (Proposal):** صياغة الفكرة كمسودة قرار معماري (ADR Draft).
3. **المراجعة (Review):** فحص المقترح ضد `QEOS_CONSTITUTION.md`.
4. **الاعتماد والتوثيق (Approval & Documentation):** الموافقة على الـ ADR ونقله إلى حالة (Accepted).
5. **[بوابة التنفيذ - Implementation Gate]** (توقف إجباري).
6. **التنفيذ (Implementation):** كتابة الكود داخل الطبقات العشرين المحددة.
7. **[بوابة الإغلاق - Closure Gate]** (توقف إجباري للتحقق).
8. **الدمج (Integration):** دمج الكود في النظام.

---

## 5. مخطط تدفق اتخاذ القرار (Decision Workflow)

```text
[New Feature / Change Request]
       │
       ▼
(Does it require a new Layer or break the 20-Layer Architecture?)
       │
       ├─► [YES] ──► REJECTED (Requires Constitutional Amendment)
       │
       ▼
      [NO]
       │
       ▼
(Does it alter an existing architectural pattern?)
       │
       ├─► [YES] ──► Write ADR ──► Review ──► Approve
       │
       ▼
      [NO] (Standard Domain Logic)
       │
       ▼
[ Implementation Gate ]
       │
       ▼
[ Write Code ]
       │
       ▼
[ Closure Gate ]
       │
       ▼
[ Merge to Master ]
```

---

## 6. نقطة السماح بكتابة الكود لأول مرة

**يُمنع كتابة أي سطر برمجي** لقرار معماري أو ميزة جديدة إلا **بعد اجتياز "بوابة التنفيذ" (Implementation Gate)** واعتماد التوثيق في سجل الـ ADR أو موافقته الصريحة للمواصفات الحالية الموجودة في مجلد `architecture/specification/`.

---

## 7. قواعد إضافة أي وثيقة مستقبلية

1. **المنع الدستوري:** لا يُسمح بإنشاء أي وثيقة تحمل صفة "Constitution" أو "Manifesto" حاكم.
2. **القرارات المعمارية:** يجب أن تُضاف بصيغة قوالب الـ ADR في مجلد `docs/architecture/adr/` حصراً.
3. **التسجيل الإلزامي:** أي وثيقة هيكلية جديدة يجب أن تُسجل أولاً في `QEOS_SOVEREIGN_REGISTRY.md` ليُعترف بها.

---

## 8. تعريف بوابات التحكم (Control Gates)

### أ. بوابة التنفيذ (Implementation Gate)
هي النقطة الفاصلة التي **تمنع بدء أي تطوير (كتابة كود)** قبل استيفاء الشروط:
- الفكرة لا تخالف `QEOS_CONSTITUTION.md`.
- القرار التقني موثق في ADR (إذا كان يتطلب ذلك).
- الطبقات (Layers) التي سيمسها التعديل محددة بدقة.
- لا يوجد أي استدعاء عكسي (Bottom-Up Call) في التصميم المقترح.

### ب. بوابة الإغلاق (Closure Gate)
هي النقطة الفاصلة التي **تمنع إنهاء أي مهمة أو دمج الكود** قبل التحقق الكامل:
- تم تنفيذ بروتوكول التحقق للوظيفة أو موجة الهجرة الحالية.
- الكود المكتوب يحترم حرفياً حدود الطبقات العشرين ولا يتخطى البوابات (Gateways).
- لم يتم تكرار أي كيانات (Entities) أو خدمات (Services).
- صدور تقرير مراجعة يثبت التوافق 100% مع الدستور.
